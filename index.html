Frequently Asked Questions

1. What is the different between var, let and const?
var: Function/global scope, can be re-declared and reassigned.
let: Block scope, can be reassigned but not re-declared.
const: Block scope, cannot be reassigned or re-declared, but contents of objects/arrays can be changed.
2. What is the different between map(), forEach(), and filter()
map(): Creates a new array by transforming each element.
forEach(): Executes a function on each element, with no return value.
filter(): Creates a new array with elements that pass a condition.
3. What are arrow function, and how are they different from regular function
Arrow function: A shorter syntax for writing functions, using =>, and it doesn't have its own this, arguments, or super.
Difference: Arrow functions are more concise and do not bind their own this, while regular functions have their own this and can be used as constructors.
4. How do JavaScript promise work
A Promise in JavaScript represents an asynchronous operation, which can be in one of three states: pending, resolved (successful), or
rejected (failed). You create it with new Promise(), using resolve for success and reject for failure. Use .then() to handle success and .catch() for failure.
5. How does closure work in JavaScript
A closure in JavaScript occurs when a function retains access to its lexical scope, even after the outer function has
finished executing. This means inner functions can access variables from the outer function, even if the outer function has returned.
